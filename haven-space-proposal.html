The Challenge

Problem Statement

The boarding house industry remains fragmented and inefficient, relying on outdated processes that create friction for all stakeholders.

  • Fragmented Discovery: Boarders struggle to find verified, quality accommodations across scattered listings
  • Manual Processes: Landlords manage bookings, payments, and communications through disconnected tools
  • Lack of Transparency: No standardized system for utilities, payments, or lease agreements
  • Trust Deficit: Limited verification mechanisms create uncertainty for both parties
  • Scalability Barriers: Existing solutions cannot support multi-property, multi-tenant operations

The Platform

Our Solution

Haven Space delivers a unified, multi-tenant monorepo platform that serves all stakeholders through purpose-built interfaces, connected by a robust, type-safe API infrastructure.

  • Public Marketplace: Discover and explore verified boarding house listings
  • Boarder Dashboard: Manage bookings, payments, and communications
  • Landlord Portal: Oversee properties, tenants, and earnings
  • Admin Console: Platform-wide oversight and user management

🏗️ Architecture Highlights

  • Next.js 16 with React 18
  • tRPC v11 for type-safe APIs
  • Prisma ORM with PostgreSQL
  • NextAuth.js v5 for authentication
  • Redis caching layer
  • Docker containerization

📦 Monorepo Structure

  • 5 applications (Public, Boarder, Landlord, Admin, Server)
  • 10 shared packages (API, Auth, Database, UI, etc.)
  • Turborepo build orchestration
  • Bun package manager
